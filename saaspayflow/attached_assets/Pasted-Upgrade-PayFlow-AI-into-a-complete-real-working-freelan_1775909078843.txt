Upgrade PayFlow AI into a complete, real working freelancer–client system.

Follow this implementation strictly in order. Do NOT skip steps.

---

STEP 1 — DATABASE STRUCTURE

Create the following tables:

1. invoice_links

* id
* invoice_id
* token
* public_url
* created_at

2. invoice_messages

* id
* invoice_id
* sender ("client" | "freelancer")
* message
* created_at

3. invoice_payments

* id
* invoice_id
* amount
* method ("upi" | "bank" | "cash" | "card" | "other")
* reference (optional)
* status ("pending" | "confirmed")
* created_at

4. invoice_requests

* id
* invoice_id
* type ("extension" | "dispute")
* reason
* message
* status ("pending" | "approved" | "rejected")
* created_at

5. invoice_activity

* id
* invoice_id
* type ("created" | "sent" | "reminder" | "payment_requested" | "payment_confirmed" | "message" | "request")
* data (JSON or text)
* created_at

---

STEP 2 — INVOICE SEND FLOW

When user clicks "Send Invoice":

* generate unique token
* create public URL (/invoice/:token)
* store in invoice_links
* insert activity: type = "sent"

Ensure:

* freelancer can always re-copy this link later

---

STEP 3 — CLIENT PORTAL CONNECTION

Inside /invoice/:token:

Enable:

1. Messaging

* save message to invoice_messages
* sender = "client"
* reflect in freelancer dashboard
* log activity "message"

2. Payment Confirmation

* client enters:
  amount
  method
  reference

* save to invoice_payments with status = "pending"

* log activity "payment_requested"

3. Extension Request

* save in invoice_requests
* log activity "request"

   * show pending payments
   * button: "Confirm payment"
     → updates status = confirmed
     → mark invoice paid (if full)
     → log activity "payment_confirmed"

3. Extension requests:

   * list all events in order
